# 注册和登录系统说明

## 系统已修复的问题

### 1. 数据库触发器
- **问题**: 注册后用户数据没有自动创建到profiles表
- **解决**: 创建了`handle_new_user()`触发器函数，当用户在auth.users表注册时，自动在profiles表创建对应记录

### 2. Profiles表结构
- **添加字段**: `full_name` (用户全名，可选)
- **现有字段**:
  - `id` (UUID, 主键, 关联auth.users)
  - `email` (文本, 必填)
  - `avatar_url` (文本, 头像URL)
  - `created_at` (时间戳)
  - `updated_at` (时间戳)

### 3. RLS权限策略
已设置以下Row Level Security策略：

**普通用户**:
- 可以查看自己的profile
- 可以更新自己的profile
- 可以插入自己的profile

**管理员**:
- 可以查看所有用户的profiles
- 可以删除任何用户的profile

### 4. 登录验证
更新了登录逻辑：
- 验证用户凭据
- 检查用户是否在profiles表中存在
- 提供更友好的错误提示（如果未注册会提醒用户先注册）

## 完整流程

### 注册流程
1. 用户在前端填写注册表单（邮箱、密码、头像）
2. 调用 `supabase.auth.signUp()` 创建认证用户
3. 触发器自动在profiles表创建对应记录（包含email）
4. 如果用户上传了头像，更新avatar_url字段
5. 注册成功，跳转到登录页

### 登录流程
1. 用户输入邮箱和密码
2. 调用 `supabase.auth.signInWithPassword()` 验证
3. 验证成功后，检查profiles表是否有对应记录
4. 如果profiles记录不存在，登出并提示用户
5. 登录成功，跳转到首页

### 后台管理
1. 管理员登录后台系统
2. 进入"客户管理"页面
3. 可以看到所有注册用户的列表（来自profiles表）
4. 显示用户邮箱、头像、预约次数、注册时间等信息
5. 可以删除用户（会同时删除认证记录和profile记录）

## 数据库关系

```
auth.users (Supabase认证表)
    ↓ (触发器: on_auth_user_created)
profiles (客户信息表)
    ↓ (外键: user_id)
bookings (预约记录表)
```

2. **登录测试**:

## 注意事项

- 触发器在用户注册时自动执行，无需手动干预
- 删除用户会同时删除认证和profile记录（级联删除）
- 所有数据访问都受RLS保护，确保数据安全
